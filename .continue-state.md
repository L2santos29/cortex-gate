## Key Components

### Gateway (axum :18801)
- Routes: `GET /health`, `GET /v1/models`, `POST /v1/chat/completions`, `GET/POST /admin/config`
- Auth: `require_client_auth` / `require_admin_auth` (HeaderMap-based, standalone functions)
- Providers: Multi-provider proxy (OpenAI, Anthropic) with tool_call ↔ tool_use conversion
- Streaming: SSE via reqwest `bytes_stream()` → mpsc::channel(64) → futures unfold → axum Sse
- Anthropic SSE → OpenAI chunks translation via state machine

### Classifier (ONNX embeddings)
- `ClassifierEngine` — `async classify(prompt) -> ClassificationResult`
- `OnnxEngine` — wraps `ort` crate v2.0.0-rc.12, 384-dim embeddings
- `RoutingEquation` — model profile scoring, confidence levels
- `Ecualizador` — visual equalizer with per-dimension knobs + economy knob
- `DimensionType` — 8 dimensions for prompt classification
- Tokenizer: WordPiece BERT vocab (30522 tokens), max 512 tokens

### Governance
- User management + API key generation
- Usage tracking with `estimate_tokens()`
- Quota enforcement
- Alert system
- SQLite via rusqlite bundled

### Benchmark Engine
- Autonomous model benchmarking
- Capability profiles by dimension
- Scorer and test suite
